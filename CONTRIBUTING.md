# Development Rules

The following rules apply to all contributions in this project:

- **Prefer small, incremental changes**: Large PRs are harder to review and more prone to errors.
- **Do not add dependencies unless asked**: Keep the project lightweight and avoid unnecessary bloat.
- **Follow existing formatting and style**: Maintain consistency with the current codebase.
- **Always explain trade-offs**: When making design decisions, document the pros and cons.
- **Suggest tests when adding logic**: Every new feature or fix should ideally be accompanied by tests.
- **Ask before refactoring large areas**: Consult with the team before making sweeping changes to existing code.
